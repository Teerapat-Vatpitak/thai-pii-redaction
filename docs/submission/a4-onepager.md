## [หัวกระดาษ] โลโก้ + AI Guard
**AI Guard — ระบบปกปิดข้อมูลส่วนบุคคลภาษาไทยก่อนใช้งาน AI ภายนอก**
ปกปิด PII ไทยก่อนส่งให้ ChatGPT/Claude แล้วคืนค่าจริงในเครื่อง — ข้อมูลจริงไม่ออกนอกเครื่อง ตาม PDPA
ระดับ Prototype · PSU Future Tech Challenge 2026

## ปัญหา
ผู้ใช้พิมพ์/วางข้อมูลส่วนบุคคล (ชื่อ เลขบัตรประชาชน เบอร์ บัญชี ประวัติสุขภาพ) ลงใน Generative AI เพื่อให้ช่วยงานเอกสาร
โดยไม่รู้ว่าข้อมูลถูกส่งออกนอกองค์กรไปเซิร์ฟเวอร์ต่างประเทศทันที เรียกคืนไม่ได้ และเสี่ยงผิด PDPA โดยเฉพาะข้อมูลอ่อนไหวมาตรา 26

## โซลูชัน (ทำงานอย่างไร — 3 ขั้น)
1. **ปกปิด (Mask):** ตรวจจับ PII ในเครื่อง แล้วแทนด้วย token `[ชื่อ_1]` หรือข้อมูลปลอมสมจริง
2. **ส่งให้ AI:** AI ภายนอกเห็นแค่ข้อความที่ปกปิดแล้ว (ไม่เคยเห็นข้อมูลจริง)
3. **คืนค่า (Restore):** สลับ token กลับเป็นค่าจริงในเครื่องจาก vault ที่อยู่ในหน่วยความจำ

ใช้งานผ่าน **ส่วนขยายเบราว์เซอร์ (Chrome Extension)** บน ChatGPT/Claude โดยตรง + รองรับ **redact PDF** (ดำกล่องทับ PII ในไฟล์)

## การใช้ AI
- Format-Preserving (กฎ + checksum): เลขบัตร (mod-11), เบอร์, อีเมล, บัญชี, บัตรเครดิต (Luhn), พาสปอร์ต, ทะเบียนรถ ฯลฯ
- NER ภาษาไทย (PyThaiNLP thainer-CRF) จับชื่อ/ที่อยู่ + กฎบริบทคำนำหน้าเสริม recall
- ธงข้อมูลอ่อนไหวมาตรา 26 (8 หมวด) + ออปชัน semantic (MiniLM) จับประโยคอิสระ
- ทั้งหมดรันในเครื่อง — ขั้นตรวจจับไม่ส่ง PII จริงให้ LLM ใด ๆ

## [แผนภาพ] สถาปัตยกรรม "สมองเดียว หลายหน้าร้าน"
```
            ผู้ใช้บน ChatGPT/Claude
                     |
        [Browser Extension]  --(ข้อความที่ปกปิด)-->  AI ภายนอก
                     |                                    |
                     v            (token เท่านั้น)        |
        Local Backend (ในเครื่อง)  <---(คำตอบ + token)----+
          - ตรวจจับ PII (กฎ + NER)
          - Vault token<->ค่าจริง (ในหน่วยความจำ)
          - คืนค่าจริง / redact PDF
```

## ผลลัพธ์ (Prototype)
- ลดความเสี่ยงข้อมูลรั่วสู่ AI ภายนอก ~100% สำหรับ PII ที่ตรวจจับได้ (vault ไม่ออกนอกเครื่อง)
- ครอบคลุม PII โครงสร้าง 10+ ประเภท + ชื่อ/ที่อยู่ + ข้อมูลอ่อนไหว 8 หมวด
- ทดสอบจริงครบลูปบน Claude สำเร็จ; ชุดทดสอบอัตโนมัติผ่าน 233 เคส

## Governance (PDPA / Responsible AI)
Vault ในหน่วยความจำเท่านั้น (ไม่เขียนดิสก์/ไม่ส่งเครือข่าย) · guard สแกนซ้ำก่อนส่ง · anti-hallucination (รับเฉพาะ span ในต้นฉบับ) · audit log ไม่มี PII · explainable · ระบุข้อจำกัดอคติ NER ตามจริง
