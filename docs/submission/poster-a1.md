## [โซน 1 — หัวโปสเตอร์ (บนสุด)]
- โลโก้ AI Guard (ใหญ่ ซ้ายบน)

- บรรทัดรอง: **ปกปิดข้อมูลส่วนบุคคลภาษาไทยก่อนใช้งาน AI ภายนอก ตาม PDPA**
- แท็ก: PSU Future Tech Challenge 2026 · ระดับ Prototype
- ชื่อทีม/สังกัด `[เติม]`

## [โซน 2 — ปัญหา (แถบสีเน้น)]
หัวข้อใหญ่: **"ข้อมูลของคุณรั่วโดยไม่รู้ตัว"**
- ไอคอน: คนพิมพ์ PII ลง ChatGPT -> ลูกศรชี้ออกนอกประเทศ
- ข้อความสั้น: "ทุกครั้งที่พิมพ์ชื่อ เลขบัตรประชาชน เบอร์ หรือประวัติสุขภาพลงใน AI สาธารณะ ข้อมูลถูกส่งออกนอกองค์กรทันที เรียกคืนไม่ได้ และเสี่ยงผิด PDPA"

## [โซน 3 — วิธีแก้ 3 ขั้น (ภาพใหญ่กลางโปสเตอร์ — จุดเด่นสุด)]
หัวข้อ: **"ปกปิด -> ส่ง -> คืนค่า"** (ในเครื่องคุณ)
แสดงเป็น 3 การ์ดเรียงกัน พร้อมภาพหน้าจอจริง:
1. **Mask** — ภาพช่องแชต Claude: ก่อน = "สมชาย ใจดี 081-234-5678" / หลัง = "[ชื่อ_1] [โทรศัพท์_1]"
2. **Send** — ภาพ: AI เห็นแค่ token (มีป้าย "AI ไม่เคยเห็นข้อมูลจริง")
3. **Restore** — ภาพ overlay คืนค่าจริงกลับมา
(ใช้ภาพหน้าจอเดโมจริงบน Claude — ของจริง น่าเชื่อถือกว่าภาพวาด)

## [โซน 4 — เบื้องหลังทำงานอย่างไร (แผนภาพสถาปัตยกรรม)]
หัวข้อ: **"ประมวลผลในเครื่อง — ข้อมูลจริงไม่ออกนอกเครื่อง"**
```
ผู้ใช้ (ChatGPT/Claude)
   |  [Extension]
   v
Local Backend (ในเครื่อง)
   - ตรวจจับ PII: กฎ+checksum (เลขบัตร mod-11/เบอร์/อีเมล) + NER ไทย (ชื่อ/ที่อยู่)
   - Vault token<->ค่าจริง (ในหน่วยความจำ ไม่เขียนดิสก์)
   --(ส่งเฉพาะ token)--> AI ภายนอก --(คำตอบ)--> คืนค่าจริงในเครื่อง
```
ป้ายเล็ก: "ไม่ส่ง PII จริงให้ AI คลาวด์ในขั้นตรวจจับ"

## [โซน 5 — ฟีเจอร์เด่น (ไอคอน 4-6 ช่อง)]
- ปกปิด 2 โหมด: token / ข้อมูลปลอมสมจริง (checksum ถูก)
- รองรับ ChatGPT + Claude ผ่าน Extension
- Redact PDF (ดำกล่องทับ PII จริง)
- ตรวจข้อมูลอ่อนไหวมาตรา 26 (8 หมวด)
- รันออฟไลน์ได้ (.exe) / on-prem
- คะแนนความเสี่ยงการระบุตัวตน (re-identification risk)

## [โซน 6 — PDPA / Governance (จุดแข็ง)]
หัวข้อ: **"ออกแบบตาม PDPA และ Responsible AI"**
- Vault ในหน่วยความจำเท่านั้น
- Anti-hallucination: รับเฉพาะข้อความที่อยู่ในต้นฉบับจริง
- Audit log ไม่เก็บข้อมูลจริง · อธิบายได้ (explainable)
- รองรับข้อมูลอ่อนไหวมาตรา 26

## [โซน 7 — ผล + ต่อยอด + ติดต่อ (ล่างสุด)]
- ผล: ลดความเสี่ยงรั่ว ~100% สำหรับ PII ที่ตรวจจับได้ · PII โครงสร้าง 10+ ประเภท · ทดสอบจริงบน Claude · ทดสอบอัตโนมัติ 233 เคส
- กลุ่มเป้าหมาย: โรงพยาบาล / ราชการ / มหาวิทยาลัย
- ต่อยอด: WangchanBERTa, OCR, redact PDF เต็มรูปแบบ
- **QR code** -> GitHub / วิดีโอเดโม · ชื่อทีม/อีเมลติดต่อ `[เติม]`
