## Full optional environment

A separate clean Python `3.13.12` environment installed core, web, ML, and OCR
extras together. It had one OpenCV owner (`opencv-contrib-python 4.10.0.84`) and
passed `uv pip check`. The full suite passed with no skipped or expected-failure
tests: **787 passed, 0 skipped, 0 xfailed**. Focused OCR/PDF and semantic/NER
suites also passed, and a real Thai PaddleOCR inference returned non-empty Thai
text with mean confidence `0.9847`.

Windows load order matters when Torch and PaddleOCR coexist: Torch must load
first. The production availability path already primes Torch, and a regression
test now pins this behavior. The reverse order was reproduced as a Windows DLL
load failure and is not treated as a supported entry path.
