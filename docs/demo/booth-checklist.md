# คู่มือหน้าบูธ AI Guard

เอกสารนี้ใช้หน้างานจริงในวันจัดบูธ อ้างอิงตาม workstream B ของ
`docs/superpowers/specs/2026-07-05-presentation-release-readiness-design.md`

## ก่อนเปิดบูธ

1. รัน preflight ก่อนทุกครั้ง

   ```powershell
   ./scripts/demo_check.ps1
   ```

   ถ้าพอร์ต 8000 ว่าง สคริปต์จะรายงาน PASS และบอกว่าแอปจะสร้าง backend เองตอนเปิด
   ถ้าพอร์ต 8000 ถูกใช้อยู่แล้วโดย backend ที่ตอบ `/api/health` เป็น ok สคริปต์จะ PASS เช่นกัน
   (แปลว่ามี AI Guard ทำงานอยู่ก่อนแล้ว) แต่ถ้ามีโปรแกรมอื่นถือพอร์ตนี้อยู่จะ FAIL
   ให้ปิดโปรแกรมนั้นก่อน หรือรันสคริปต์พร้อม `-KillOrphans` เพื่อเคลียร์โปรเซส `aiguard*`
   ที่ค้างอยู่

2. เปิดแอปล่วงหน้าอย่างน้อย 1-2 นาทีก่อนแขกมาถึงจุดแรก เพื่อให้ backend อุ่นเครื่องแล้ว
   ตัวแอปมีเพดานรอสูงสุด 30 วินาที (60 ครั้ง x 500 มิลลิวินาที ต่อการเช็ก `/api/health`)
   ระหว่างนั้นจะเห็นข้อความ "กำลังเริ่มบริการในเครื่อง..." บนหน้าจอ ห้ามให้แขกเห็นหน้านี้
   ระหว่างสาธิตสด เปิดแอปทิ้งไว้ตั้งแต่ก่อนเปิดบูธจะช่วยตัดความเสี่ยงนี้ไปเลย

3. โหลด extension แบบ unpacked ใน Chrome ผ่าน `chrome://extensions` เปิด Developer mode
   แล้ว Load unpacked เลือกโฟลเดอร์ `extension/` และล็อกอิน claude.ai ไว้ล่วงหน้า
   (ทดสอบว่า mask/restore ทำงานได้จริงบนหน้า claude.ai ก่อนเปิดบูธ)

4. ปิดการแจ้งเตือนของ Windows และปิด sleep/screen timeout ระหว่างจัดบูธ
   (Settings > System > Notifications ปิดทั้งหมด, Settings > Power ตั้ง never sleep)

5. เชื่อมต่อ hotspot จากโทรศัพท์ไว้เป็นสำรอง ทดสอบว่าเน็ตใช้ได้จริงก่อนเปิดบูธ

## Run-of-show ประมาณ 3 นาที

1. **สาธิตข้อความบนหน้าจอ**: mask ข้อความตัวอย่างที่มี PII ไทย ส่งให้ AI ภายนอก
   แล้ว restore กลับ ทำทั้งสองโหมด
   - token mode: PII ถูกแทนด้วย token เช่น `[ชื่อ_1]`
   - surrogate mode: PII ถูกแทนด้วยข้อมูลปลอมที่อ่านลื่นเหมือนข้อความจริง

2. **สาธิต hotkey**: กด Ctrl+Shift+M เพื่อ mask ข้อความที่คัดลอกไว้ใน clipboard
   แล้วกด Ctrl+Shift+R เพื่อ restore กลับ ทำให้เห็นว่าใช้ได้แม้ไม่เปิดหน้าแอป

3. **สาธิต PDF redact**: อัปโหลดเอกสารตัวอย่าง แสดงภาพก่อน/หลัง redact แบบ black-box
   ที่ตำแหน่ง bbox จริง

4. **สาธิตรายงานความเสี่ยง**: เปิดหน้า risk report ให้ดูคะแนนความเสี่ยง PDPA
   และรายการ Section 26 ที่ถูก flag (ย้ำว่าเป็นการ flag เพื่อแจ้งเตือนเท่านั้น
   ไม่ใช่การ redact อัตโนมัติ)

## Drill กู้สถานการณ์

| สถานการณ์ | วิธีแก้ |
|---|---|
| backend ตาย (แอปค้าง ไม่ตอบสนอง) | ปิดแอปทั้งหมด แล้วเปิดใหม่ รอจนขึ้นหน้าใช้งาน (เวลาที่วัดได้จริงจาก `scripts/measure_demo.py` ให้กรอกในตารางด้านล่าง) |
| wifi ที่บูธตาย | สลับไปใช้ hotspot จากโทรศัพท์ที่เตรียมไว้ |
| hotspot ก็ตายด้วย (ไม่มีเน็ตเลย) | ใช้ desktop loop แบบออฟไลน์: mask/restore ข้อความและ redact PDF ทำงานได้ทั้งหมดโดยไม่ต้องต่อเน็ต เพราะโมเดล NER ฝังมากับตัวติดตั้งแล้ว งดขั้นตอนที่ต้องส่งให้ AI ภายนอกจริง (เช่นส่งไป claude.ai) แต่ยังโชว์ mask -> restore แบบ local ได้เต็มรูปแบบ |

## Fallback ladder (ทางเลือกสุดท้ายเรียงจากดีที่สุด)

1. สาธิตผ่าน extension บน claude.ai (เต็มรูปแบบ ต้องมีเน็ต)
2. สาธิตแบบ desktop loop ออฟไลน์ (mask/restore + PDF redact เท่านั้น ไม่ต้องมีเน็ต)
3. เปิดวิดีโอสาธิตความยาวไม่เกิน 5 นาทีที่ส่งไปแล้วในรอบสมัคร
4. โชว์ภาพหน้าจอ (screenshot) ที่เตรียมไว้ใน USB

## ตัวเลขที่วัดได้ (กรอกจากผลของ scripts/measure_demo.py)

รันคำสั่งนี้ตอน backend อุ่นเครื่องแล้ว (ไม่ใช่ cold start) เพื่อวัด latency จริง:

```powershell
$env:PYTHONUTF8='1'; .\.venv\Scripts\python.exe scripts\measure_demo.py
```

| รายการ | ค่าที่วัดได้ (ms) | หมายเหตุ |
|---|---|---|
| GET /api/health | _กรอก_ | |
| POST /api/sanitize (token) | _กรอก_ | |
| POST /api/sanitize (surrogate) | _กรอก_ | |
| POST /api/reidentify | _กรอก_ | |
| POST /api/redact-pdf | _กรอก_ | ใช้ `examples/sample_document.pdf` |
| sanitize(token) + reidentify รวมกัน | _กรอก_ | สคริปต์จะเตือนถ้าเกิน 2000 ms |
| cold start (เปิดแอปครั้งแรก ถึงหน้าใช้งาน) | _กรอก_ | เพดานสูงสุดของโค้ดคือ 30 วินาที (60 x 500ms) จับเวลาจริงด้วยนาฬิกา |

## หมายเหตุสำหรับขั้นตอนที่ต้องทำบนเครื่องจริงเท่านั้น (B4 / B5)

ขั้นตอนสองข้อนี้ทำแบบ headless ไม่ได้ ต้องทำบนเครื่องที่จะใช้จัดบูธจริง
ด้วยตัวติดตั้งจริง (ไม่ใช่ dev environment) และต้องล็อกอิน claude.ai จริง

### B4: ซ้อมแบบออฟไลน์เต็มรูปแบบ

1. ตั้งเครื่องเป็น airplane mode (ปิด wifi และ bluetooth ทั้งหมด)
2. เปิดแอป AI Guard ที่ติดตั้งจริง ยืนยันว่าขึ้นหน้าใช้งานได้โดยไม่มีเน็ต
3. ทดสอบ mask -> restore ข้อความ ยืนยันว่าโมเดล NER ที่ฝังมากับ .exe ทำงานได้ครบ
4. ทดสอบ redact PDF ด้วย `examples/sample_document.pdf` ยืนยันว่า black-box วางถูกตำแหน่ง
5. บันทึกผลว่าผ่านหรือไม่ผ่าน พร้อมเวลาที่ใช้

### B5: ถ่ายภาพหน้าจอความละเอียดสูง (ต้องกว้าง >= 3000 พิกเซล)

ภาพเหล่านี้เป็น hard dependency ของโปสเตอร์ A1 (C3) ต้องถ่ายบนเครื่องจริง
พร้อมล็อกอิน claude.ai จริง ไม่สามารถประกอบขึ้นเองแบบ headless ได้

ขั้นตอน:

1. ตั้งความละเอียดจอแสดงผลเป็นระดับสูงสุดที่จอรองรับ (Settings > Display)
   ถ้าจอกว้างไม่ถึง 3000 พิกเซล ให้ปรับ scaling ของ Windows ลง (เช่น 100%)
   เพื่อให้พื้นที่ capture จริงกว้างขึ้น หรือใช้เครื่องมือ crop มาต่อกันหลังถ่าย
2. เปิด claude.ai ในเบราว์เซอร์ พิมพ์หรือวางข้อความตัวอย่างที่มี PII ไทย
3. ถ่ายภาพ **ก่อน mask**: หน้าข้อความต้นฉบับที่ยังมี PII เต็ม
4. กด mask (ปุ่มในหน้า extension หรือ Ctrl+Shift+M) ถ่ายภาพ **หลัง mask**:
   ข้อความที่ถูกแทนด้วย token/surrogate แล้ว ก่อนกดส่ง
5. ส่งข้อความให้ Claude ตอบกลับ กด restore (Ctrl+Shift+R) ถ่ายภาพ **restored overlay**:
   คำตอบของ Claude ที่ถูกแทน token กลับเป็นข้อมูลจริงแล้ว
6. เปิดหน้า redact PDF ถ่ายภาพ **PDF ก่อน redact** (เอกสารต้นฉบับ)
   และ **PDF หลัง redact** (มี black-box ปิดทับ PII)
7. ใช้เครื่องมือ Snipping Tool (Win+Shift+S) หรือ screenshot เต็มจอ (PrtScn)
   แล้วครอปเฉพาะส่วนที่ต้องการด้วยโปรแกรมแต่งภาพ หลีกเลี่ยงการย่อขนาดภาพหลังถ่าย
8. บันทึกไฟล์เป็น PNG (ไม่ใช่ JPEG) เพื่อไม่ให้ตัวหนังสือไทยแตกเป็น artifact
9. ตรวจสอบความกว้างไฟล์จริงด้วย คลิกขวา > Properties > Details ต้อง >= 3000 พิกเซล
   ก่อนส่งต่อให้ workstream C ไปใช้ในโปสเตอร์
